# Diskurs Content App

Многостраничное веб-приложение на GitHub Pages + Google Apps Script API.

## Структура файлов

```
diskurs-app/
├── index.html                  ← страница входа
├── src/
│   ├── styles.css              ← общие стили
│   ├── utils/
│   │   ├── config.js           ← URL API (ЗАПОЛНИТЬ!)
│   │   └── api.js              ← клиент API + Auth + утилиты
│   └── pages/
│       ├── projects.html       ← страница 1: проекты + ввод данных
│       ├── status.html         ← страница 2: статус всех проектов
│       └── head.html           ← страница 3: детальные данные из Хэд
└── api/
    └── Code.gs                 ← бэкенд в Google Apps Script
```

---

## Шаг 1 — Настроить Apps Script

1. Открой таблицу → Расширения → Apps Script
2. Замени содержимое `Code.gs` содержимым файла `api/Code.gs`
3. Запусти функцию `createEmployeesSheet()` один раз — создаст лист «Сотрудники»
4. Замени пароли на настоящие в листе «Сотрудники»
5. **Опубликуй как веб-приложение:**
   - Развернуть → Новое развёртывание
   - Тип: Веб-приложение
   - Выполнять как: **Me**
   - Доступ: **Все**
6. Скопируй URL вида `https://script.google.com/macros/s/XXXXX/exec`

---

## Шаг 2 — Вставить URL в конфиг

Открой файл `src/utils/config.js` и замени строку:

```js
API_URL: 'https://script.google.com/macros/s/ВСТАВЬ_СВОЙ_ID/exec',
```

на свой URL из шага 1.

---

## Шаг 3 — Загрузить на GitHub Pages

1. Создай репозиторий на GitHub (например `diskurs-app`)
2. Загрузи все файлы **кроме папки `api/`** (она не нужна на GitHub)
3. Settings → Pages → Source: `main` branch, папка `/ (root)`
4. Приложение будет доступно по адресу:
   `https://ТВО_ИМЯ.github.io/diskurs-app/`

## Важно про Apps Script и CORS

Apps Script Web App поддерживает CORS при условии:
- POST-запросы отправляются с `Content-Type: text/plain` (не `application/json`)
- Приложение опубликовано с доступом «Все»

Именно так настроен `api.js` — не меняй заголовки запросов.
